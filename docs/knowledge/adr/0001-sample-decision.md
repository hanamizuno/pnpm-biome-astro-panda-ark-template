# ADR-0001: サンプル — 決定タイトル

> **これはサンプル ADR です。** テンプレートに同梱しており、ADR の典型的な形を示すためのもの。最初の本物の決定で上書きするか (`0001-` を再利用)、本サンプルを参照として残すなら次から `0002-` で始めてください。不要であれば削除して構いません。

## Status

Accepted — 2026-07-11

## Context

どんな問題を解こうとしていたか。どんな力学 (技術的制約・締切・チーム体制・既存のコミットメント等) が働いていたか。何年後にこれを読む人が、当時の会話にアクセスできなくても状況を理解できるように書く。

コードを読めば自明なことの再記述は避け、**この決定を当たり前ではなく「決定」たらしめた力学** (トレードオフと制約) に焦点を当てる。

## Decision

決定を 1〜2 文で言い切ったあと、詳細を展開する:

* 採用するもの (具体的に)。
* 明示的に棄却するもの (検討した代替案と、それぞれを退けた一言理由)。
* 決定のスコープと境界。

## Consequences

この決定が **何を簡単にし、何を難しくするか**:

* 運用上の帰結 (チームが今後やるべき行動の変化)。
* 移行・ロールアウト上の影響。
* 既知のフォローアップや、この決定が意図的に先送りした未解決事項。
* 関連する ADR・convention・runbook へのクロスリンク。例: `[conventions/...](/docs/knowledge/conventions/...md)`
