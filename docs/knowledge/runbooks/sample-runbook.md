# サンプル — Runbook タイトル

> **これはサンプル runbook です。** テンプレートに同梱しており、runbook の典型的な形を示すためのもの。実際の手順で上書きするか、不要であれば削除してください。

## When to use this

このランブックが適用される **シグナル** — アラート・エラーメッセージ・顧客からの報告・しきい値超過のメトリクス等。深夜 3 時にオペレーターが目の前のシグナルとこの runbook を曖昧さなく結び付けられる粒度で書く。

## Pre-flight checks

* 本当にこの runbook が該当するかを素早く確認するチェック (見た目の似たインシデントとの切り分け)。
* 着手前に手元に揃えておくべき権限・認証情報・ツール。

## Steps

1. 最初のコマンド / アクション。具体的なコマンド行をインラインで書く。
2. 次のステップ。期待される出力や観測される変化を併記する。
3. ...

ステップは番号を振り、コマンドはインラインで載せる。明らかでない失敗モードがあるステップは、そのステップ直下で即座に注意喚起する。

## How to confirm it worked

インシデントが解消したことを示す **具体的なシグナル** (ダッシュボードのグリーンチェック、特定のログ行、スモークテストの成功等)。これがないと、オペレーターはいつ手を止めてよいか分からない。

## If it does not work

* 次に試すこと。
* エスカレーション先と判断基準。

## Postmortem hooks

* ポストモーテム用に、その場で記録しておくべき情報 (タイムスタンプ・実行コマンド・観測した副作用等)。
